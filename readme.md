# PCAP 分析器

简单的命令行 PCAP 包解析工具（支持交互式筛选和翻页）

## 迭代记录

- **迭代一**：基础解析与打印（原始包逐个解析并打印）
- **迭代二**：交互式翻页浏览（先扫描建立索引，再支持 n/p/q 翻页查看）
- **迭代三**：交互式筛选配置 + 模糊匹配支持（启动后交互式配置筛选条件，支持通配符 *）

## 模块作用与协作流程

整体流程：
```
main.c 
  ↓ 启动
交互式输入 pcap 文件名 + 配置筛选条件
  ↓ 调用
pcap_reader.c  ← 第一遍扫描建立索引
  ↓ 第二遍进入交互模式（n/p/数字键翻页）
packet_parser.c  ← 把二进制包解析成 PacketInfo
  ↓ 判断是否显示时调用
filter.c  ← 检查是否匹配筛选条件（支持通配符 *）
```

### 各个文件的具体作用

| 文件 | 作用 |
|------|------|
| **main.c** | 入口：启动后询问文件名和筛选条件，初始化筛选配置，调用 pcap_reader 先扫描后进入交互查看 |
| **packet_parser.h/c** | 解析器：负责把原始以太网/IP/TCP/UDP/ICMP 二进制数据解析成人能看懂的信息（源IP、目的IP、端口、协议等） |
| **filter.h/c** | 筛选器：拿着解析好的 PacketInfo，判断这个包是否符合用户想要看的条件（比如只看 TCP、只看某个 IP），支持通配符 * 模糊匹配，符合就返回 1，不符合就返回 0 |
| **pcap_reader.h/c** | 读取器：负责打开 pcap 文件，先扫描一遍建立索引，然后按页读取并显示，支持 n/p/数字键/q 交互，里面会调用 packet_parser 解析包，调用 filter 判断要不要显示 |
| **Makefile** | 一键编译脚本 |

### 交互式翻页操作
```
=== 第 1/3 页 (数据包 1-20/50) ===
No.     Time                Source                Destination         Protocol  Length  Info
---------------------------------------------------------------------------------------------------------
1       10:15:30.123456     192.168.1.100:54321   93.184.216.34:80    TCP       66      [SYN]
2       10:15:30.123567     93.184.216.34:80      192.168.1.100:54321 TCP       66      [SYN, ACK]
...

操作: n=下一页, p=上一页, [数字]=跳转到页, q=退出
> 
```

### 交互式按键
```
n / 空格  - 下一页
p         - 上一页
[数字]    - 跳转到指定页（例如输入 3 然后回车跳转到第 3 页）
q         - 退出
```

### 筛选条件支持的模糊匹配
筛选 IP 时可以使用通配符 `*`，例如：
- `192.168.*` - 匹配所有 192.168 开头的 IP
- `10.*` - 匹配所有 10 开头的 IP
- `*` - 匹配所有 IP（等同于不设置）
